# Contributing to Quizora

First off, thank you for considering contributing to Quizora! It's people like you that make Quizora such a great tool for learners and competitors.

## Getting Started

1. **Fork the repository** on GitHub.
2. **Clone your fork** locally.
3. **Install dependencies** using `npm install`.
4. **Create a branch** for your feature or bugfix (`git checkout -b feature/your-feature-name`).
5. **Make your changes**. Please ensure your code adheres to our Neo-Brutalist design language where applicable.
6. **Test your changes** by running `npm run dev` and verifying locally.
7. **Commit your changes** with descriptive commit messages.
8. **Push to your fork** and submit a **Pull Request**.

## Pull Request Guidelines

- Ensure any new UI components utilize the `.neo-btn` or `.neo-card` Tailwind utilities.
- Do not commit any API keys (e.g., Gemini).
- Make sure `npm run lint` passes without errors.

## Code of Conduct

Please note that this project is released with a Contributor Code of Conduct. By participating in this project you agree to abide by its terms.
